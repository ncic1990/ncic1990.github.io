Cluster Parallel Database

Cluster Parallel Database

Director: Sun Ninghui
Time: June,2003——December,2004

    Along with the proceeding of informationlization which represented by education and E-government, database is more required for scalability, including the data capacity and the request number with customer. The deficiency of system and database resources usually appears in the running process of an application system. The products represented by Oracle OPS parallel database , aims at high-end affairs such as bank, telecommunication, etc. They focus on increasing the affair processing ability and single query speed. These products are expensive and out of accordance with many current needs of applications. This project intends to avert from the strong aspect of international manufacturers , to develop parallel database in a 2-4 nodes cluster in the foundation of domestic database DBMS , to satisfy the demand to the scalability of database for special industry as education and E-government. Customer could increase resources constantly. The new system could satisfy the demand of the increasing data capacity and the increasing request number of customers.

　　This project is a sub-project of Key Science and Technologies Program of Beijing City " large general database management system development". Data and knowledge engineering institute, Renmin University of China do the research of parallel database. The research of cluster related technique is processed by NCIC, ICT, CAS. The key problem in research has three aspect. First, keep the data distribution with consistency and data integrality , combining with the dynamic scalability. Second, satisfy the high speed correspondence that required by parallel database. Last, High Available Software in parallel database.

Technique strategy:
(1) Together with the development of cluster, research will be conduct on how to support parallel database on 2-4 nodes cluster, including storage system, OS optimization, optimization of customized high speed data channel, HA technique etc. Marketplace and technique innovation will be found beyond the limitation of this solution.
(2) The Parallel Database does not emphasis on improving transaction-processing capacity and speed of single query. Application mode should be limited a little. Firstly, Master/Slave architecture is used while master node serves for all requests and processes meta data; secondly, Share-Nothing mode, high speed data channel used among nodes; lastly, based on indigenous database of Kingbase, user applications need little modification.
(3) Roll mode is used to support multiply parallel database semantics and meet different application modes.

Goal:
(1) Cluster system is consist of four nodes and each node is configured with Cpu:2*2.4GHz, 2G memory, RAID storage 1TB , customized high speed network and G-bit Ethernet, IP protocol is supported.
(2) A suite of TPC-C performance evaluation program for Parallel Database
(3) A suite of Parallel database for 4-node including data distribution, query processing distribution, HA technique for Parallel Database, database dynamic extension etc; Three Parallel database application modes. 1: full distribution of database tables, every node has full database; 2: some tables are shared and replication among nodes, guarantee of data consistency; 3: a lot of tables are shared like DB-link.

Proceeding:

Achievement:
(1): Connection Manager.
(2): Operation-oriented replication server and performance measurement.
(3): Research on transaction-oriented replication

Ongoing work:
Transaction-oriented replication;

Future work:
(1) Database recovery
(2) Data segmentation and distributed query and optimization.
