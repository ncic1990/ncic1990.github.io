The Dawning Cluster File System

    DCFS is a cluster file system developed by NCIC (National Research Center forIntelligent Computing Systems, Institute of Computing Technology, ChineseAcademy of Sciences) for Linux clusters, especially the Dawning 4000LSuper-server.

    DCFS is a global file system shared by all cluster nodes. Applications see asingle name space from all nodes.

    DCFS provides standard interface for users. Files in DCFS can be accessedthrough OS system calls and can be manipulated by system commands such as ls,cp, tar, vi, etc. Applications based on traditional UNIX file I/O can use DCFSfiles without recompilation.

    DCFS exploits a scalable architecture. It is able to run on large clusters.Metadata processing is divorced from file data processing in DCFS. They havedifferent servers and different access path. Moreover,there can be multipleservers for both file data and metadata.

    File data are stored on multiple storage servers in a RAID 0 style ofstriping. And metadata are stored on and maintained by multiple metadataservers. Both metadata and file data are stored in regular files of nativefile system (EXT2) on servers.

    DCFS is able to provide both high aggregate I/O bandwidth and high metadataperformance.

    DCFS exploits a flexible communication mechanism, which enables it to use highperformance communication protocols such as BCL, VIA, etc, besides TCP andUDP.

    DCFS provides a set of management utilities. Their functions include DCFSsystem configuration, starting and stopping DCFS services, and mount andunmount DCFS file system.
