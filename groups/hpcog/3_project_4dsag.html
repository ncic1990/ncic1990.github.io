项目简介

DSAG OS是支持网格化动态自组织体系结构(Dynamic Self-organized Architecture based on Grid-component)的一种新型操作系统。所谓DSAG是一种新的高性能计算机体系结构，简单的说，DSAG就是根据应用的计算模式和运行模式的需要，自动利用高速网络中独立的、网格化的功能部件动态组织成一个计算机系统，并有效运行应用程序。

传统计算机操作系统是一种与节点计算机绑定的小而全模式：操作系统只负责本地的资源分配，而不同计算机节点之间的协调则通过应用软件进行。在DSAG结构中，组成计算机的功能部件可以独立的上网并提供服务，传统意义上的本地计算机节点已经不存在，相应的本地操作系统也将不复存在。

网格化的计算机将是一个利用动态组合的资源构成的一个全局的计算机系统。在DSAG OS中，相应传统单机操作系统的作业管理、内存管理、文件系统、I/O处理等等必须在全局的角度重新进行考虑。DSAG OS的特征包括：

  • 操作系统功能的分布化
  • 适合应用模式的按需组合的资源映射
  • 资源的动态申请、分配和组合
  • 实现单一系统映像,隐藏系统结构差异
  • 自动故障屏蔽和恢复

 

我们对 DSAG OS的研究基于DSAG模拟器SandFox进行，SandFox不仅包括对硬件体系，还包括对操作系统的模拟。SandFox的结构如下图所示：

SandFox结构图

 

SandFox包括：

  • 由多个SandMP组成的CPU Box
  • Memory Box，即Remote Memory
  • 外设，比如网卡Internet Gateway
  • 高速互连部件RC－Switch（比如光互连），上述资源通过此部件进行高速互连
  • 系统管理网络Net Switch：上述部件都有MNET接口，资源通过这些接口接入Net Switch，从而与系统管理软件进行交互。通过Net Switch，资源可以动态加入或者退出；同时管理软件可以通过Mnet Switch向相应部件发出控制信息，比如进行系统配置等等
  • DSAG OS：对资源进行动态管理、分配和配置，处理与用户的交互，完成用户提交的任务

 

DSAG OS包括：

  • DSAG Manager：对资源进行管理，支持资源的动态增减、发布和协商；根据用户需求分配资源，通过Mnet Switch配置出相应的系统，加载SandPOS，并支持运行时的动态资源管理。
  • SandPOS：通过远程系统调用机制构建Linux环境，二进制兼容Linux应用，并构建简单高效的计算节点操作系统SandPOS。SandPOS支持单处理器、SMP、CC-NUMA和MPP等体系，同时还需提供一定的机制支持运行时资源的动态申请和释放。
  • 远程系统调用服务器SandRServer，即Remote OS Service：SandPOS支持对进程和存储的有效管理，但是没有本地文件系统，同时设备管理也非常简单；应用程序的文件操作利用远程系统调用机制通过请求SandRServer完成。远程系统调用服务器体现了操作系统功能的分布化，同时有利于实现单一系统映像。

项目进展

2005.7.27 863软件专项“支持网格化的动态自组织计算机体系结构的操作系统”通过专家组验收

2005.6 SandPOS完成资源发布和动态配置等功能，DSAG OS原型系统完成。

2005.1  在SandPOS上成功移植了openmp编译器ompi0.81和Nas NPB2.3 –omp。

2005.1  Sand OS作为SandUPSim的一部分对外开放源码。

2004.12  对SandPOS的MPI库进行增强，并成功移植了Nas npb2.3-mpi。

2004.12  利用远程系统调用机制实现了基于SandFox的MPP系统节点间rfork和透明的节点间rpipe通信，此原型系统用于验证远程系统调用可以用于实现单一系统映像。

2004.11  Sand OS取得国家版权局计算机软件著作权登记证书。

2004.9  Sand OS通过SPEC CPU2000 train和ref数据集测试，系统比较稳定。

2004.8.19  DSAG OS项目接受863项目办中期检查。

2004.8  完成SandPOS对MPP的支持，并开发了SandPOS基本MPI库。

2004.7  开始设计开发MPP版本的SandPOS。

2004.6  完成SandPOS对SMP的支持，并实现了SandPOS基本pthread库。

2004.5  开始设计开发SMP版本的Sand OS ，Sand OS因此升级成 SandPOS。

2004.4.1  在863软件重大专项操作系统研讨会上报告了DSAG OS的研究进展情况。

2004.3  完成对DSAG OS初步的总体设计，与此同时，并行版本模拟器SandFox也开始设计开发。

2004.3  Sand OS可以加载动态链接的Linux ELF程序。

2003.12  Sand OS通过SPEC CPU2000 test数据集测试。

2003.10  Sand OS基本功能已经具备，Linux应用程序环境基本建立，可以加载运行静态链接的Linux ELF程序。

2003.7  863软件专项“支持网格化的动态自组织计算机体系结构的操作系统”正式启动，此操作系统称为DSAG OS。配合单机模拟器SandUPSim的开发，DSAG OS的初期目标为单机操作系统Sand OS。

2003.5  提出“网格化的动态自组织的计算机体系结构”(DSAG)的思想
