     About

 

1. Introduction

----------------

  SandUPSim is a unique processor simulator which is developed by Institute of Computing Technology (ICT) of China Academy of Sciences. It is dense computing oriented and binary compatible with the Linux program.

 

  SandUPSim includes the simulation of hardware architecture and the operating system. It could be divided into three parts: SandMips, Sand OS and Remote Server. SandMips is the simulation of hardware. Sand OS and Remote Server are the system software.

 

Structure and Linux application environment of SandUPSim

 

  SandUPSim can be used for architecture research, hardware validation, system software debugging, arithmetic analysis, software analysis, system optimization, fault injection and analysis, system simulation and education, etc.

 

2. Our Work

------------

  SandMips is based on vmips, an open source C++ simulator based on a MIPS R3000 processor. We add the components such as CPU pipe line, floating point coprocessor, Cache, DDR simulation, Virtual Service, network card and performance Stat., etc. We also enhance the debugging function, make some changes to the structure of the simulator and optimize the performance. SandMips is a full system simulator which simulates CPU, Cache, memory, system bus and peripheral devices. It is also a multi-granularity, flexible, simple, well-structured and scalable simulator.

 

  We design and implement a light-weight operating system over SandMips which is called Sand OS. Sand OS is based on Topsy, a small teachable micor-kernel operating system developed by Swiss Federal Institute of Technology. It contains the following main modules: boot strap, memory management, thread management and device management, and doesn't have the local file system. It is simple and effective. We enhance the functions of Topsy thoroughly. We also add Linux system call API, program loader with Linux ABI compatibility, communication protocol based on TCP/IP and Shell to build Sand OS.

 

  Sand OS supports Linux ELF programs with binary compatibility. Because it has no local file system, the file operating system calls are implemented by requesting the Remote Server. This server is developed completely by us.

 

  In system test, SandUPSim overpasses lots of Linux programs, such as SPEC CPU2000. It shows that the simulator has good Linux application compatibility and could be used to support many researches in the computer architecture field.

 

3. License

-----------

  The three parts of SandUPSim is relatively independent. Because the license of vmips is different from that of Topsy, we authorize the three parts individually. The licenses of vmips and Remote Server are both GPL, and the license of Sand OS is still that of Topsy.

     History

 

Jan/2005

    SandUPSim V1.0 is now available! This is the first time we release SandUPSim as an open source simulator.

 

 Nov/2004

    We got a software enrollment for SandUPSim V1.0.

 

 Sep/2004

    SandUPSim ran SPEC CPU2000 with the train and ref data sets successfully. It shows that SandUPSim is relatively stable.

 

 Mar/2004

    SandUPSim supports the Linux dynamic linking ELF programs.

 

 Dec/2003

    SandUPSim ran SPEC CPU2000 with the test data set successfully.

 

 Oct/2003

    The Linux application environment of SandUPSim was setup. But it was not stable yet.

 

 Jun/2003

    We launched the SandUPSim project.
