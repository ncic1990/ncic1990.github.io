PVM的使用

设置环境变量

一.1000A

若用户使用的是 C shell，要确认下列内容已在你的 $HOME/.cshrc 文件中：

　　setenv PVM_ROOT /usr/local/pvm3
　　setenv PVM_ARCH RS6K

　　并分别在MANPATH和path中加入 $PVM_ROOT/man和$PVM_ROOT/lib两路径。

若用户使用的是 k shell, 则需确认下列内容已在你的 $HOME/.profile 文件中：

　　export PVM_ROOT=/usr/local/pvm3
　　export PVM_ARCH=RS6K

　　并分别在MANPATH和PATH中加入 $PVM_ROOT/man和$PVM_ROOT/lib 两路径。

二.2000-I及3000

CSH TCSH环境下，检查以下的代码是否在.cshrc中：
　　if ( -f /cluster/config/sys.conf ) then
　　@ net={ grep NET /cluster/config/sys.conf >& /dev/null }
　　if ($net) then
　　setenv PVM_ROOT /cluster/pvm/net
　　setenv PVM_ARCH RS6K
　　else
　　@ mesh={ grep MESH /cluster/config/sys.conf >& /dev/null }
　　if ($mesh) then
　　setenv PVM_ROOT /cluster/pvm/mesh
　　setenv PVM_ARCH D3K
　　endif
　　endif
　　else
　　echo "/cluster/config/sys.conf is not found"
　　endif

BSH,KSH环境下：检查以下的代码是否在.profile中：
　　if ( -f /cluster/config/sys.conf ) then
　　@ net={ grep NET /cluster/config/sys.conf >& /dev/null }
　　if ($net) then
　　export PVM_ROOT /cluster/pvm/net
　　export PVM_ARCH RS6K
　　else
　　@ mesh={ grep MESH /cluster/config/sys.conf >& /dev/null }
　　if ($mesh) then
　　export PVM_ROOT /cluster/pvm/mesh
　　export PVM_ARCH D3K
　　endif
　　endif
　　else
　　echo "/cluster/config/sys.conf is not found"
　　endif

[注]：通常系统管理员已为你设置好以上环境。
 

启动PVM

一.在计算节点启动PVM

　　用户应在node1 - node8节点启动PVM，这要根据你当时所能分到的节点而定。在你所允许登录的任一计算节点上，在unix提示符下键入命令：

　　　　%pvm

　　若返回pvm控制台提示符为：pvm>，则 pvm 已在此节点启动，然后你即可按pvm控制台命令进行加入所需节点等操作。

　　pvm 的另一启动方式是使用命令：

　　　　%pvm pvmfile

　　其中文件pvmfile的内容和功能请参看PVM的主机文件。


二.启动时可能遇到的问题

　　若PVM启动有问题，则会在计算机终端或在log文件 /tmp/pvml.<uid> 中给出有关信息。启动时常遇到的一个问题可能是：

　　[t80040000]Can't start pvmd

　　这往往是由于上次运行PVM非正常结束而引起，因为它可能会遗留下 /tmp/pvmd.<uid>文件，这一文件仅应在PVM运行时存在。若启动前存在，则会阻止PVM的启动，将其删除，问题会解决。
 

PVM控制台命令

1.add 后面跟随一至数个机器名，指示PVM系统将它们加入到当前的虚拟机中，命令执行完毕时控制台返回成功地加入到虚拟机中的节点机台数和节点机名称。如果返回的数字不等于机器名的个数，则表明有些节点加入失败。如：

　　　　pvm>add node1
　　　　1 successful
　　　　HOST DTID
　　　　node1 80000
　　　　pvm>

　　说明了节点加入成功。如果使用计算网，相应命令是：

　　　　pvm>add hnode1

2.conf 显示当前虚拟机的配置，包括虚拟机中所有节点机的机器名、运算速度、节点机类型等。

3.quit 退出PVM控制台,返回shell提示符。退出后PVM daemon仍在运行,在控制台中所进行的一切设置仍然有效。

4.help 将给出 pvm 控制台下所有可用命令的使用。

5.help [command] 将给出 [command] 的详细使用说明。

6.halt 清除当前虚拟机中的所有应用程序并终止当前虚拟机。

 

PVM的主机文件

　　控制台程序允许用户在命令行上给出一个文件名，文件中包含一串机器名，控制台自动将这些机器加入到虚拟机中，该文件称为主机文件 (hostfile)，此文件（文件名任取）每行给出一个机器名，机器名之后还可给出一些选项。使用主机文件启动PVM的命令格式见启动PVM 。此文件中常用的选项为：ep=path给出该节点机上用户的PVM可执行程序所在的目录。缺省值为：$HOME/pvm3/bin/$PVM_ARCH。若用户未在hostfile中给出ep项， 则需由用户在其主目录下建立子目录pvm3/bin/RS6K，并将PVM应用程序的目标文件置于该子目录下，否则在运行应用程序时会返回错误信息。为方便使用，用户可利用ep项给出PVM执行码所在目录。

　　一般地，hostfile文件中的空行及以字符“#”开始的行均被控制台忽略。

　　hostfile的一个例子如下：

　　　　# hostfile
　　　　node1 ep=/public/soft/user1/src
　　　　node2 ep=/public/soft/user1/src
　　　　.
　　　　.

　　如果使用计算网，则 hostfile 格式如下：

　　　　# hostfile
　　　　hnode1 ep=/public/soft/user1/src
　　　　hnode2 ep=/public/soft/user1/src
　　　　.
　　　　.
 

PVM程序的编译

一.1000A

1.Include文件

　　PVM提供了一个供C语言使用的头文件 $PVM_ROOT/include/pvm3.h 和一个供Fortran语言使用的头文件 $PVM_ROOT/include/fpvm3.h，它们定义了PVM函数库中所需的全局变量和常数。它们的使用方式如下：

C语言：

　　在文件头使用 # include <pvm3.h>，并在编译时使用 -I$(PVM_ROOT)/include 。

Fortran语言：

　　需给出全路径名，如：include '/usr/local/pvm3/include/fpvm3.h'，也可用链接方式做，即在源程序文件所在目录下，在unix提示符下做：% ln –sf /usr/local/pvm3/include/fpvm3.h fpvm3.h，这样用户在其Fortran源程序中可直接使用include 'fpvm3.h'，而不必使用全路径名。

2.编译PVM应用程序

　　对于已写好的PVM应用程序，可使用以下四种方法之一进行编译。

　　（1）% pvmf77 *.f (Fortran程序)
　　（2）% pvmcc *.c (C语言程序)

　　事实上，pvmf77和pvmcc是为方便用户而写的两个shell命令文件，驻留于目录/usr/local/pvm3/lib之下，用户可直接调用来编译已有的Fortran或C 语言应用程序。他们的使用方式类似于一般的cc和f77编译器。

　　（3）make

　　与一般unix系统的make使用方法相同，只需在make文件中加以适当的包含文件和库函数链接并设置适当的路径（需链接的函数库和必需的路径见上述说明）。

　　（4）aimk

　　为方便应用程序的编译和链接，PVM系统提供了一个实用程序 (Architecture Independeut Make)。aimk也是一个shell命令文件（见目录 /usr/local/pvm3/lib下相应文件），它具有与标准unix的 make 程序类似的功能， 能自动为用户设置ARCHLIB和PVM_ARCH的值，缺省情况下的aimk使用的make文件是Makefile.aimk。在 /usr/local/pvm3/examples 子目录下有文件Makefile.aimk的一个例子。用户可按需要将此文件拷到自己的目录下，并加以修改使用。用aimk编译程序的命令格式为：

　　 　　% aimk -f flname pss

　　其中flname为make文件名，缺省时为Makefile.aimk或makefile，pss为所生成的执行文件名。

　　对初学者，使用pvmf77（或pvmcc）常常可满足编译一般程序的要求，特别是源程序只包含少数文件的情况。

PVM程序的链接

一、网络版（net)

1.C语言

　　链接:
　　$(PVM_ROOT)/lib/$(PVM_ARCH)/libpvm3.a, 即：
　　/usr/local/pvm3/lib/RS6K/libpvm3.a

2.Fortran语言

　　除上述PVM库外还要链接：
　　$(PVM_ROOT)/lib/$(PVM_ARCH)/libfpvm3.a，即：
　　/usr/local/pvm3/lib/RS6K/libfpvm3.a和
　　/usr/local/pvm3/lib/RS6K/libpvm3.a

　　若用到了组功能，还需链接：
　　$(PVM_ROOT)/lib/$(PVM_ARCH)/libgpvm3.a, 即：
　　/usr/local/pvm3/lib/RS6K/libgpvm3.a

二、MESH版（以下是曙光3000上的PVM系统所需要的环境）

1.C语言

　　需要链接以下的库:
　　-L$(PVM_ROOT)/lib/$(PVM_ARCH) -L$(MPI_ROOT)/lib -L/cluster/bcl/lib -L/cluster/rms/lib -L/cluster/sdr/lib　-lpvm -lmpich -lbcl -lrms -lsdr

2.Fortran语言

　　除上述PVM库外还要链接：
　　$(PVM_ROOT)/lib/$(PVM_ARCH)/libfpvm3.a

　　如果用户使用曙光系统提供的pvmf77和pvmcc的话，就不需要在自己的编译环境下链接以上的库。
 

PVM程序的运行

一.1000A

　　在曙光天潮1000A系统上，PVM应用程序一般在节点node1 - node8运行， 这要根据系统所能分配的资源决定, 其步骤如下:

1. 登录到节点机

　　从主机（如carrot）的unix环境下，rsh到任一个你可登录的节点机, 如node1: % rsh node1。在node1 适当的子目录下启动PVM。

　　a. 在 $HOME/pvm3/bin/RS6K子目录下，只需用下述命令启动PVM：

　　　　　　% pvm

　　b. 在用户的任一非a所述的子目录下，或者直接构成虚拟机系统，则需用下述命令启动PVM（其中hostfile文件内容见PVM的主机文件）：

　　　　　　% pvm hostfile

　　　　构成虚拟机。

　　　　在PVM提示符 pvm> 下使用pvm控制台命令 (add) 加入所需节点机（当然也可以在hostfile中给出），并用控制台命令

　　　　(conf) 确认虚拟机结构正确。

2. 退出虚拟机PVM界面

　　　　% pvm> quit

　　正常情况下系统会给出信息　pvmd still runnig，并返回unix提示符。

3. 运行PVM应用程序

　　在适当目录下，在unix提示符下键入执行程序名，如 % hello

4.PVM环境的退出

　　若PVM应用程序结束运行，用户应用以下方式正常退出PVM环境，首先在 unix 提示符下键入以下命令：

　　　　% pvm

　　则系统返回pvm提示符并给出有关信息，一般是：

　　　　pvmd already running
　　　　pvm>

　　在此提示符下键入命令halt：

　　　　pvm>halt

　　则清除当前虚拟机中的所有应用程序，且pvm正常退出。

二.2000-I及3000

　　在2000-I及3000下加载模式与1000A一样，同时又提供新的加载模式：pvmrun

　　　　% pvmrun [[-sz] [size of node]] [[-np] [size of proc]] [[-pn] [node list]] [[-cpu] [0/1]] <program [usaer argv]>

例子:

　　在两个节点上加载4个用户进程－－hello,cpu的0为缺省定义。

　　　　%>pvmrun -sz 2 -np 4 -cpu 0 hello

　　在指定节点0和节点2上加载4个用户进程－－hello,cpu的1是指每个节点都要完全的使用所有的cpu。

　　　　%>pvmrun -pn 0,2 -np 4-cpu 1 hello
