面向网格的机群系统管理软件的研究与实现


人们对计算机运算速度和处理能力的要求总是没有止境的，对于有些需要超高性能的计算机应用领域，如科学与工程计算、决策支持和事务处理等商业计算、大型信息服务和网络服务、数字图书馆、生物信息处理和视频多媒体系统等，传统的大型计算机（Mainframe）和全对称多处理系统（SMP）越来越难于满足这些领域不断增长的应用需求。基于这一认识，国家智能计算机研究开发中心研制了曙光3000超级服务器，它是一种通用的可扩展超级并行计算机，能够以较好的性能价格比满足这些应用的需求。

随着网格技术的发展和它将成为未来并行计算的主流趋势，曙光4000超级服务器的研究将定位为面向网格的超级服务器，为了支持这种服务器，服务器的系统软件也必须面向网格，作为机群系统软件集的一个重要的组成部分，系统管理软件也必须面向网格。本文主要研究的是面向网格的机群系统管理软件的实现机制。
系统管理软件是机群系统管理员管理和监控整个机群状态的有力工具。曙光4000超级服务器是曙光3000的延续，仍然采用机群机构，因此，它的系统管理软件在功能和基本遵循原则上没有太大的变化，变化的是它的体系结构和实现机制。本文首先研究了基本的机群系统管理软件设计路线、遵循原则和基本功能，在此基础上设计实现了三种不同体系结构的系统管理软件。这些系统管理软件在基本思想和基本设计原则上一脉相承，但是由于不同的结构和考虑，采用了不同的实现语言和通讯机制。本文重点研究了面向网格的机群系统管理软件的设计和实现，同时本文对这三种系统管理软件在实现语言和通讯机制这两方面进行了详细的性能分析和评价并给出了测试数据来辅助说明。最后通过这些性能测试和分析对面向网格的机群系统管理软件做出客观的评价。

关键词：系统管理，机群，网格，体系结构

Research and Implementation of grid-oriented Cluster System Management Software

People's request for the processing speed and efficiency of computers is limitless. In the fields of science and engineering computing, commercial computing such as decision-making and transaction processing, cosmic information service and network service, digital library, biology information processing and video multimedia system, which need super efficient performance to deal with, traditional Mainframe computer and SMP become less satisfied with the greatly increasing request. Based on this acknowledge, National Research Center for Intelligence Computing System, Chinese Academy of Science developed a kind of superserver which was named for "dawning 3000". The super server is a kind of all-purpose parallel-computing server and is extendable. It satisfies the needs of these applications with providing better rate of performance and price.

With the development of Gird technology and the current of becoming main trend of parallel-computing in the future, "Dawning 4000 Superserver" will be developed to a kind of superserver which faces grid. In order to support the grid superserver, the cluster operation system of it must face grid. Because Cluster System Management Software is an important component of cluster operation system, it must face grid and provide service to grid. The content of my thesis is researching the development of a kind of cluster system management software which faces grid.

Cluster system management software is useful for the cluster system Manager to control and monitor the status of whole cluster. "Dawning 4000 Superserver" is based on "Dawning3000 Superserver" and has the same cluster structure, so the system management software of it will has few change in basal functions and essential way of design principle. What change is system structure and realization of it. The thesis first analyses fundamental way of designing the cluster system management software, principles which must be followed and basal functions. Then based on the analysis, study and realize three kinds of system management softwares which have different system structure in detail. The three softwares are similar in the basic thinking and principle of designing, but due to different structure and considering, they are developed by different programming languages and communication system. The stress of the thesis is the analysis and realization of the cluster system management software of facing grid. In the thesis, I provide detail analysis to the performance of the three softwares with detail testing data and evaluate them. At last, through these analysis and testing data, I make an objective evaluation to the cluster system management software of facing grid.
