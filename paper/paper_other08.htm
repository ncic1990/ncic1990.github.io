高效并行计算系统中的计算模型与计算网络


随着计算机应用技术的高速发展，人们对计算机的处理速度的要求也日益提高。而机群技术是当前提高计算机系统运算性能的主流技术。在机群系统中，由于每个计算结点需要频繁的交换大量的数据，所以通信子系统的通信性能就成了影响整个系统性能的关键之一，故而设计一个高效、易用、功能丰富的通信系统是提高机群系统整体性能的重要途径。

由于当前主流机群系统中，作为计算结点的计算机系统都是SMP 结点，每台结点机器上有多路CPU，这在提高了结点计算能力的同时也提高对于高速通信的性能要求。在传统的单网卡/单通路的机群系统中，由于网络硬件的物理限制，机群系统的通信性能成为了整体性能的瓶颈。为了突破这种限制，人们提出了使用多网卡/多网络通路进行并行通信的解决方法。在曙光3000 超级服务器机群系统的通信子系统中，我们加入了对并行通信的支持，有效的改善了系统的通信带宽。

本文中，在对当前通信子系统的细致分析的基础上，详细讨论了各种并行通信实现的设计模式，进而分析了在实现中的各种关键技术。随后对于我们的并行通信实现做了详细的介绍，包括对系统网络拓扑的图示、对设计总体思想的描述、几个关键算法的展示以及该实现所特有的创新之处。经过分别在我们搭建的两个测试用结点群上的测试后，我们得到了各种网络性能的测试数据，包括网络带宽和通信延迟。通过分析我们可以看到，在该系统上的并行通信实现基本达到了预期的性能目标，虽然由于机器物理配置的限制影响了部分数据。同时，本文中也提出了对当前并行通信实现的优化方法与改进思路。

Research and Realization of Parallel Communication On Cluster
Abstract

With the rapid development of computer applications, the request for the processing and efficiency of computers became more and more urgent. Today, cluster has become the main technology to accelerate the computer system. In clusters, every node machine needs to exchange a lot of data with the other nodes, so the efficiency of communication subsystem is a key factor of the whole efficiency of cluster system. It is a very important to design a efficient, powerful communication subsystem, and should be easy to use.

Because of the high processing speed of the node machines, which are usually SMPs, the ability of the communication subsystem has become a bottle-neck of the cluster. And the restriction comes from the limit of network device. To resolve this problem, a new method has been put forward. That is parallel communication mechanism, which uses multi-netcard or multi-network to send and receive message. On Dawning-3000 Cluster, we added the implement of this mechanism into its communication subsystem to increase the bandwidth of its network.

In this paper, I discussed all the probably design patterns and selected one from them as ours technology route. And I have analyzed the key factors of the realization of parallel communication. In the following of this paper, the details of this software module were introduced, particularly, the differences from the other implements were pointed out. In the end of this paper, I listed the result of the tests of our software on two different clusters and the analysis of them.
