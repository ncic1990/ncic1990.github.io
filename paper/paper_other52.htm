机群系统配置管理的研究与实现

摘　　要

机群就是通过高性能网络或局部网络将一组计算机系统（结点）互联，形成的具有单一系统映像的高性能、高可用性、高扩展性的计算机机群系统。由于机群系统结构松散、结点独立性强、网络连接复杂，造成机群系统管理不便，难以使用。为了解决这个问题，国际上流行的方式是在各个结点机操作系统之上再建一层操作系统来管理整个机群，称之为机群操作系统。本文以曙光4000机群系统为科研背景，以机群操作系统中的配置管理为具体对象，进行了广泛而深入的研究。

曙光4000机群操作系统是一个集成的、一体化的机群中间件系统，机群系统配置管理软件是这个中间件系统的一个重要组成部分，我们称之为机群操作系统的一个“服务”。该服务是从原有的机群系统软件中抽取出来的可以共享的服务之一，它负责管理机群系统的配置信息，为机群基础构件及机群应用层构件提供主动智能的信息服务。机群系统配置管理软件的设计采用了基于服务和一体化构件的思想，以基于CORBA的分布式构件方式实现，具有主动信息服务的特性、良好的可扩展性、高可用性和系统的包容性。

论文首先介绍的是机群、中间件、分布式构件及主动信息服务的概念和发展状况等相关内容，其次，文章对机群系统配置管理软件的设计思想和采用的关键技术进行了研究和分析，再次，论文重点描述了机群系统配置管理软件在体系结构和功能上的设计和实现，并为配置管理软件给出了客观的评价，最后，文章论述了作者所参与的对高可用机群内核自主引导方法的进一步研究。

关键词：机群，配置管理，服务，构件，自主引导

 

Research and Implementation of

a Configuration Management System for Clusters

Abstract

Cluster is a system which is connected with a group of node computers via high performance network or local area network and provides high performance, high availability, high scalability. The usage of cluster system is difficult because of incompact structure, strongly independent nodes and complex network. In order to solve this problem, the popular method is to build operating system on nodes’ operating systems, called cluster operating system. This dissertation is based on research of Dawning 4000 and summarizes research in the Configuration Management System(CFMS) for Clusters.

The “Dawning 4000” cluster operating system is an integrated middleware system on clusters. Configuration Management System belongs to it and is called a service. Being taken out from the past cluster system software, this service can be shared by different system components . The Configuration Management System is useful for managing and providing cluster configuration information on its own initiative. The author adopts the idea of service-based design and component-based development to design the CFMS and implement it with CORBA. The CFMS has the characteristic of high availability service assurance, extensibility, compatibility, automated functional conversion and so on.

First, this dissertation introduces the conceptions and related contents of clusters, middleware, components and active information service; then, designing ideas and key techniques will be analyzed and researched; after that, this dissertation describes the implementation of architecture and function in details; then, it gives an objective evaluation of the Configuration Management System; at last, this dissertation summarizes the author’s research in the cluster kernel’s self-boot methodology.

Key Words：Cluster，Configuration Management，Service，Component，Self-boot
