1. Overview

The Dawning-1000 Massively Parallel Processing System, which is the most important key project of the 863 Program, has been successfully developed by the National Research Center for Intelligent Computing Systems (NCIC) in April, 1995. Its success indicates that the technologies of the massively parallel processing (MPP) of China have come up to the advanced world levles, and its success is considered as a milestone of the high performance computer community of China.

The Dawning-1000 broke through some key technologies in the MPP, and exploits a lot of advanced technologies of 1990s. Its peak speed achieves 2.5 GFLOPS (billion floating point instructions per second), and its practical computing speed is 1.58 GFLOPS. It has up to 1024 Mbytes of memory. The Dawning-1000 represents the most advanced high-performance computer in China. It is a efficient, stable and reliable system which is easy to use. Its technical features includes:

  • Good extensibility, include scalable connection network, expandable computing power and I/O power. Both its hardware and software are scalable. It can scale up to a MPP system containing hundries or even thousands of computing nodes. It obviously excels the SMP systems and the tranditional vector machines in this point.
  • Large number of nodes, high computing power, large volume of memory. It can handle the problems which are hard to deal with by workstations or traditional supercompter. For examples, a weather forecast program, which forecasts the coming 48 hours weather condition of the whole area of China, needs only 3 hours to run on the Dawning-1000, but runs over 60 hours on ordinary computers such as workstations (the results are of no use then). A 3-dimention iterative deviation kernel program for seismic data processing in the petroleum prospecting field runs only 11 minutes on the Dawning-1000, but runs over 10 hours on workstations without results. Compares with vector supercomputers, the Dawning-1000 has higher performance/cost ratio.
  • The self-designed, stable and reliable wormhole communication chip, which extensively exploits the combination of asynchrous mode and synchrous mode, with higher message passing speed.
  • The message passing protocol with flow control, which provided high efficiency and high reliability for system communications.
  • High quality of the optimizing compiler, which provided efficient and practical parallel compile and comes up to the advanced world level.
  • the advanced parallel programming environment, parallel debugging tools, automatic parallelizing tools and the realtime visualization tool of parallel programs, including PVM, EXPRESS, NX Interface, Autopar, NDB, and Paravision, etc., make the use of the MPP system much easier.
  • Standard and high quality of the system design and the hardware production, high quality of the system software, high reliability of the whole system.
  • Extensively exploiting the international standards, the highly open system, which makes it much easier for the users to port the environments, tools, and large application softwares.
The Dawning-1000 comes up to advanced world level of the early 1990s.

The Dawning-1000 can be widely used in many fields such as seismic data processing in the petroleum prospecting field, weather forecast, scientific research, teaching and education, business processing, banking, stock broking, and so on, it can bring about tremendous economic and social benefits. Especially, It is the ideal choice for the computer system of supercomputing centers.

2. Main Technological Targets of the Dawning-1000

  • Node number: total 36 nodes, among which there are 32 computing nodes (the CPU is i860xr), 2 service nodes and 2 I/O nodes.
  • Peak computing speed: 2.56 GFLOPS for single precision and 1.92 GFLOPS for double precision.
  • Memory volume: 1 GB, 32 MB per node.
  • Harddisk volume: over 5 GB.
  • Exploiting the wormhole 2D mesh communication network, with the 4.8 GB/s total communication capacity.
  • The bandwidth between the computing nodes and the communication network is 2.8 Gb/s.
  • Running parallel programs of C, C++ and FORTRAN, providing the parallel optimizing reconstruction tool PORT and the automatic parallelizing tool Autopar, and providing optimization for source codes and object codes on a single computing node.
  • The UNIX based distributed parallel operating system.
  • The parallel program debugging environment NDB.
  • Partial of the large software for the parallel digital weather forecast and partial of the large software for seismic data processing in the petroleum prospecting field.
3. Hardware of the Dawning-1000
3.1 System Architecture
The Dawning-1000 is a loosely-coupled message passing massively parallel processing system. It is made up of 32 computing nodes (each of which is based on the i860 microprocessor), the system service subsystem which contains one or two nodes, and the external storage subsystem which contains two or three I/O nodes. All these nodes are connected by a 6x6 2-dimention mesh. Each point on the mesh has a wormhole router. There is a system monitor which charges the system initialization and the hardware failure diagnosis. The working platforms for the users are SUN Sparc-II workstations (or other workstations, eg. SGI) and some terminals. The system servic nodes, monitor, the Sparc-II workstations and the terminals are connected by a 10 Mb/s Ethernet. Fig 1 shows the Dawning-1000 system architecture.
3.2 Mesh Network
The mesh network in Fig 1 is extensible. Each node (including system service node and I/O node) is connected with a wormhole routing chip (WRC). The numbering of the nodes (the physical addresses of the nodes) is made up of two parts: the numbering in X direction and the numbering in Y direction. The relative address between two nodes is contained in messages. A WRC has 10 input/output channels. Each channel contains 8 datalines, a Tail signal which indicates the end of a message, a sending request line (REQ) and an acknowledging line (ACK). When a message is sent from a source node to a destination node, it only passes through the WRCs on the way, and doesn't go into the node processors connected by the WRCs. The mesh used X-Y routing algorithm. Several channels can share a unidirectional physical channel.

3.3 Computing Nodes
A computing node is a plug-in board, the kernel of which is a i860xr chip, the frequency of the chip is 40 MHZ, the word length is 64 bits, the floating point computing speed is 80 MFLOPS for single precision and 60 MFLOPS for double precision, the scalar computing speed is 40 MIPS. It has 32 MB DRAM with parity check, 8 KB EPROM, and 2 KB NVRAM which is powered by batteries. The node-mesh interface module contains two 16 KB FIFOs. The communication bandwidth between the mesh and each node is 33 MB/s, with CRC check of message data. The RS232 module on each node board is directly connected to a multi-user card on the monitor in order that the monitor can supervise and disgnose the node processor. All computing nodes run operating system and user programs.

3.4 System Service Nodes
The system service nodes are provided for managing the system resources (especially the node processors). A service node is a 50 MHZ 486 PC with a EISA bus. There is a EISA-Mesh interface module (EMI) between the EISA bus and the mesh network. There is DMA mechanism on the EMI card, which is a EISA plug-in card. The data path FIFO-Mesh-FIFO has CRC check. The communication speed between the EMI card and the mesh is 16 MB/s. When there are a lot of users and a lot of graphic displaying, two service nodes can be provided, and they can work coordinately by shareing a system resource table.

3.5 I/O nodes
An I/O node is a 486 PC with a EISA bus, as the same as the service nodes, and has a EMI plug-in card. An I/O node can be connected to a 4 GB harddisk by a SCSI card pluged on the EISA bus. The system can have one or two I/O nodes. User who require large I/O throughputs can have more I/O nodes, but the number of computing nodes must be reduced correspondingly. The data transfering speed between an I/O node and the mesh network is 16 MB/s. Each I/O node runs a parallel file system, and performs parallel file operations.

3.6 System Console
There is a system console for the whole system, which runs UNIX. The console has a multi-user card, which is connected to the RS232 serial ports of the 32 computing node boards. The console can perform point-to-point communication with any computing node, and also can broadcast messages to all computing nodes simultaneously. Its main fuctions are initializing the system, supervising the running of the operating system and user programs on the computing nodes, dinamically looking up the node processor resource configuration table and diagnosing the hardware failures of the computing nodes.

3.7 Network of the Front-End Computers
The working platforms for users of Dawning-1000 are one or more SUN Sparc-II colored graphical workstations (or SGI graphical workstations), and some terminals. All these devices are connected to the system service nodes by a LAN (Ethernet or WAN). The LAN obeys TCP/IP protocol, and its data transfering speed is 10 Mb/s. In fact, the mesh system can be considered as a computing server for the front-end network system. User programs are edited, compiled, linked and debugged on the front-end computers, and then are sent to the compuing nodes by the service nodes, and the computing results are displayed by the front-end computers.

4. Software of the Dawning-1000
4.1 Operating System
The operating system of the Dawning-1000 is based on UNIX. The operating system on the computing nodes is a micro kernel operating system, supporting virtual memory management, trap processing, job management and message passing. The file serving is a client running on the user space.

The operating system on the service nodes is based on Mach2.6, supporting multithread. Its main function is system management, especially the node processor allocation and deallocation. The I/O nodes run the parallel file server PFS. The service nodes and the I/O nodes have the same message passing interface with the computing nodes. The system implements message passing protocol with flow control, which is efficient and reliable.

4.2 Parallel Optimizing Compiler and Tools
The compile optimization for the node programs includes source code level optimization and object code level optimization. The former exploits the methods such as loops exchange, loops merging and loops strip dividing. The latter is fully utilize the pipeline, the double-instruction mode and the double-operation instructions provided by the i860, and the floating point load instructions to vectorize the loops.

The Autopar is a self-developed automatic parallelizing tool, which can recognize parallel loops, perform data partition, and produce parallel FORTRAN source codes with loop level parallel.

Paravision, the visualization tool of parallel programs is developed for the Dawning-1000, and is also portable on other MPP systems. Users can use it to watch the dynamic running of programs, including message passing conditions, load balancing conditions and whether or not deadlock occuring, etc.

4.3 Parallel Programming Environment and Parallel Debugging Tools
We have ported three well-known interface or environment ---- NX Interface, EXPRESS and PVM on the Dawning-1000, in order to facilitate users programming and running parallel programs. The NX is a parallel user programming interface on the Paragon system. The EXPRESS and PVM are two extensively used parallel programming and running environment in the world.

The parallel program debugger includes the source code level debugger NDB and the assembly code level debugger ADB. Their chief functions are display, setting and clearing instruction (or data) breakpoints, memory read/write, running programs step by step or continuously, or sending and receiving the states of current process. They can debug programs running on multiple nodes.

5. Concluding Remarks

  • The Dawning-1000 achieved ten "first" of China.

  1. The first massively parallel processing system developed all by designers in China.
  2. The fastest peak floating point computing speed in China, which is 2.5 GFLOPS.
  3. The first supercomputer made in China which practical computing speed exceeds 1.0 GFLOPS, for example, the matrix multiplication speed is 1.58 GFLOPS, and the linear equations solving speed is 1.12 GFLOPS.
  4. Solving the largest size problems that the supercomputers made in China can solve. It can solve the linear equations of order 15,000, run the 3-dimention iterative deviation program of seismic data process in the petroleum prospecting field (the amount of the computing is very large). The speedup ratio of 32 computing nodes is nearly 30.
  5. The largest memory volume, which is 1024 Mbytes.
  6. For the first time in China that we have successfully developed the practical wormhole routing chip.
  7. For the first time in China that we have successfully developed the distributed parallel operating system of our own copyright.
  8. For the first time in China that we have successfully developed the effective and practical parallel optimizing compiler for distriputed parallel computers.
  9. For the first time in China that we have comprehensively used and improved the world porpular parallel programing environments.
  10. For the first time in China that we have porting and improved the parallel debugging tool.
  • The Dawning-1000 came up to the advanced world levels of the early 1990s.

  1. The self-developed wormhole routing chip creatively exploited the combination of asynchronous mode and synchrounous mode, and it is one of the several most advanced routing chips in the world.
  2. The parallel optimizing compiler of the Dawning-1000 is assessed as one of the few most excellent systems in the world by the world famous colleague scholars like Arvind in MIT and Gao Guang-Rong in the University of McGill. The Paravision, one of the tools on the Dawning-1000, is very advanced in the world, on which the person in charge of the technologies for the Japanese Fifth Generation Machina Project spend many years but failed.
  3. The Dawning-1000 exploits the advanced architecture. Its operating systems, software environments and tools, and user interface all use the international standards. The overall level of its software is equivalent to that of cuerent MPP systems.
  4. Testings have shown that the performance of the Dawning-1000 is equal to that of famous MPP systems of the same class and scale, and on some aspects, eg., the largest problem size it can solve, it exceeds the system of the same class and scale.
